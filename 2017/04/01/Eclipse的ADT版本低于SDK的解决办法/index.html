Eclipse的ADT版本低于SDK的解决办法

发表于 2017-04-01 | 分类于 博文 | 评论：

今天手欠升级了SDK，从22升级到了23以上，而ADT版本仍为22。

存在的问题：

  1. 无法通过自动更新获取较新的ADT
  2. 无法通过离线更新的方式升级ADT，应该是冲突导致的，一篇百度经验说要卸载旧的ADT插件，链接在这里

解决办法：

而我选择了按照这篇博文说的，IT_醉仙的专栏 id : IT_大仙

直接修改SDK版本号，降为22，以匹配ADT的版本。

原文摘录如下：

其实如果只是因为sdk更新，从而工程报错，解决这个问题还有一种方法，不用更新adt，而是将sdk中的tools文件夹下的lib文件夹下的plugin.prop文件的begin plugin version版本设置低一点，比当前adt对应的sdk版本低或者一样就行！建议用这种方法！如果非要钻牛角尖 那就试试我上面说的步骤！

比如说我的路径是这样的：

E:\eclipse_64\adt-bundle-windows-x86_64-20140321\sdk\tools\lib

直接用 Notepad++打开就好了。
