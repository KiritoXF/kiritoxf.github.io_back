Java学习

所看的是Mars 的 Java4Android系列视频 ，本文是整理的笔记。如有错字可在评论中指出。

面向对象基础

先整体，后局部。面向对象是抽象的。面向对象是为了减少代码量。

Cat mew = New Cat();

重载

void funA(){};
void funA(int i){};
void funA(int i,double d){};

构造函数

Cat(){};
Cat(int i, String name);

最后更新日期：2017.03.27

this的使用方法

  1. 使用this调用成员变量和成员函数
  2. 使用this调用构造函数

this.参数;

this(参数1, 参数2);
调用本类中另一个构造函数，参数和格式分别为括号所示。

static关键字的使用

静态函数和静态变量都可以用类名调用。

静态代码块
在装载类的时候自动调用，用的较少。

  1. 静态成员变量只有一份
  2. 在静态函数当中不能使用this
  3. 静态代码块的主要作用是为静态成员变量赋值

继承初步

Java只支持单继承。继承可以减少重复代码。
在父类的基础上进行扩展。

class superCat extends Cat{
    int power;
    void fight(){
        System.out.println("我有" + this.power + "点能量");
    }
}

子类实例化&super的应用

子类若不写构造函数，则默认加一行

super();

调用父类无参构造函数，括号内的参数决定调用哪一个构造函数。

函数的复写

对继承到的方法有不满意、需要修改的时候，在子类重新定义这个方法。

条件:

  1. 在具有父子关系的两个类当中
  2. 父类和子类各有一个函数，这两个函数的定义（返回值类型、函数名和参数列表）完全相同

复写父类函数时，可用 super.函数名() 减少重复代码。

对象的转型

向上转型:

Student s = new Student();
Person p = s;

一个引用能够调用哪些成员（变量和函数），取决于这个引用的类型

Student可以study()，但是Person不可以study()

一个引用调用的是哪一个方法，取决于这个引用所指向的对象

向下转型:

Student s1 = new Student();
person p = s1;
Student s2 = (Student)p;

抽象类和抽象函数

  • 使用abstract定义的类被称之为抽象类
  • 抽象类不能够声称对象
  • 如果一个类当中包含有抽象函数，那么这个类必须被声明为抽象类
  • 如果一个类当中没有抽象函数，那么这个类也可以被声明为抽象类
  • 抽象函数就是没有函数体的函数
  • 抽象类不能生成对象，但是却可以拥有构造函数

为什么要使用抽象类？

为了防止开发人员忘写什么东西。

包和访问权限

将类放置到一个包当中，需要使用package 包名，如：

package   io.github.kiritoxf.test;

编译时：

java -d   d:\java\src   test.java;

包名的命名规范：

  1. 包名所有的字母都要小写
  2. 一般情况下，是倒置的域名段

软件包为Java类提供了命名空间

权限：

public > protected > default > private

  • public 可以修饰类、成员变量 和 成员函数

    没有任何限制，同一个包当中，不同包当中的类都可以自由访问

  • private 可以修饰成员变量 和 成员函数

    只能在本类当中使用

  • default(默认就是default) 可以修饰类、成员变量 和 成员函数

    在同一个包当中，可以自由访问

接口的基本语法

  1. 使用interface定义
  2. 接口当中的方法都是抽象方法
  3. 接口当中的方法都是public权限
  4. 一个类可以实现多个接口
  5. 一个接口可以集成多个接口

  6. 实现接口使用 implements 关键字

    class USBPhone implements USB{}

Java当中的异常

使用try~catch~finally来处理可能出现异常的代码

try{

}
catch(Exception e){
    e.printStackTrace();
}
finally{
    //关闭文件的代码
}

throws 声明一个函数可能会产生异常

Java中的I/O

I/O系统的主要目标是为了对数据进行读写操作

数据的流向以Java程序为参照物

I/O的分类

第一种分法：

  • 输入流
  • 输出流

第二种分法：

  • 字节流
  • 字符流

第三种分法：

  • 节点流
  • 处理流

I/O当中的核心类

  • InputStream - > FileInputStream

  • OutputStream - > FileOutputStream

核心类的核心方法

  • InputStream:

int read(byte [] b,int off, int len)

读取数据，读进来放进byte[]，off表示从byte[]的第几位开始放，len表示最多读多少数据。
返回值为，此次调用read方法总共读进来多少数据。

  • OutputStream:

void write(byte[]b, int off, int len)

一次性读一行数据：
public String readLine()

throwsIOException

内部类和匿名内部类

class A{
    int i;
    class B{
        int j;
        int funB(){
            int result = i + j;
            return result;
        }
    }
}

B 就是 A 的内部类了，在 B 中使用的变量 i 就是 A 中的 i

使用时是这样：

A a = new A();
A.B b = a.new B();

匿名内部类的使用：

B b = new B();
b.fun(new A(){
        public void doSomething(){
            System.out.println("xf");
        }
    });

Java当中的线程

  • 多进程：

　在操作系统中能（同时）运行多个任务（程序）

  • 多线程：

　在同一应用程序中有多个顺序流（）同时）执行

创建线程的方法：

  • 方式1：

定义一个线程类，它继承类Thread并重写其中的方法run()，方法run()称为线程体。

  • 方式2：

提供一个实现接口Runnable的类作为线程的目标对象，在初始化一个Thread类或者Thread子类的线程对象时，把目标对象传递给这个线程实例，由该目标对象提供线程体。

  1. 生成一个Runnable接口实现类的对象
  2. 生成一个Thread对象，并将Runnable接口实现类的对象传递给该Thread对象
  3. 通知Thread对象，执行start方法

线程的简单控制方法：

  • 中断线程

    • Thread.sleep()
    • Thread.yield()

  • 设置线程的优先级

    • getPriority()
    • setPriority()

线程的优先级最大是10，最小是1，可以使用Thread所提供的静态常量来设置线程的优先级。优先级越大，执行的概率越高，并不一定执行。（因为CPU抢占）。

类集框架

什么是类集框架？

  1. 类集框架是一组类和接口
  2. 位于java.util包当中
  3. 主要用户存储和管理对象
  4. 主要分为三大类-集合、列表、映射

Collection接口

  • boolean: add(Object o)
  • void: clear()
  • boolean: isEmpty()
  • remove(Object o)
  • int: size()

迭代器 Iterator

调用 Set 对象的 Iterator 方法，会生成一个迭代器对象，该对象用于遍历整个Set
import java.util.Set;
import java.util.HashSet;
import java.util.Iterator;

`Set<String> set = new HashSet<String>();`
`Iterator<String> it = set.iterator();`

Iterator > Collection > Set > HashSet

映射（Map）

映射中的每一个元素包含一个键对象和一个值对象（键值对），键不可以重复，值可以重复。

`import java.util.Map;`
`import java.util.HashMap;`

`HashMap<String, String> hashMap = new HashMap<String, String>();`
`Map<String, String> map = hashMap;`

equals 函数

equals 函数继承于 Object 函数。

什么是对象的内容相等

需要符合如下两个条件：

  1. 对象的类型相同（可以使用instanceof操作符进行比较）；

  2. 两个对象的成员变量的值完全相同。

    User u1 = new User();
User u2 = new User();
boolean b1 = u1.equals(u2);

本身的 equals 函数与连等符号 == 完全相同。

可以通过函数复写来拓展其功能。
（这不就相当于自己写了个函数吗，跟equals有啥关系）
